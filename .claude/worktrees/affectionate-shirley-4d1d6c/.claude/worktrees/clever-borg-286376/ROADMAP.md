### المرحلة 7 — توسعة القنوات والتكاملات
**الأولوية:** عالية | **التعقيد:** كبير

#### 7A — Instagram DMs
- [ ] ربط Meta Graph API لقناة Instagram
- [ ] استقبال رسائل Instagram في نفس الصندوق الوارد
- [ ] تبادل رسائل نصية + صور عبر Instagram
- [ ] مؤشر القناة (WA / IG) على كل محادثة
- [ ] إعداد Webhook منفصل لـ Instagram

#### 7B — WooCommerce
- [ ] ربط WooCommerce REST API (Consumer Key + Secret)
- [ ] جلب الطلبات غير المكتملة تلقائياً
- [ ] مزامنة حالات الطلبات مع WooCommerce
- [ ] دعم مزود متجر ثانٍ بجانب Shopify (اختيار من الإعدادات)

#### 7C — Zapier / Webhooks الصادرة
- [ ] إرسال Webhook عند كل تغيير حالة طلب
- [ ] إرسال Webhook عند وصول رسالة جديدة
- [ ] لوحة إدارة Webhooks (URL + أحداث مختارة + Secret)
- [ ] سجل طلبات Webhook (نجح/فشل + retry)

#### 7D — نقاط الولاء
- [ ] رصيد نقاط لكل عميل (مخزّن في inbox.json)
- [ ] منح نقاط عند تأكيد الطلب / الشحن
- [ ] استبدال النقاط بكوبون خصم (Shopify Discount API)
- [ ] عرض رصيد النقاط في بطاقة العميل بالصندوق الوارد
- [ ] رسالة WhatsApp تلقائية عند بلوغ حد النقاط

### المرحلة 8 — التوصيل والتوسع الحقيقي
**الأولوية:** عالية | **التعقيد:** متوسط

#### 8A — نشر السيرفر على السحابة
- [ ] ملف `Dockerfile` للباكند
- [ ] إعداد `railway.json` / `render.yaml` للنشر بضغطة
- [ ] متغيرات البيئة عبر واجهة المزود (بدون `.env` يدوي)
- [ ] دومين مخصص + SSL تلقائي

#### 8B — نشر الفرونت على Vercel
- [ ] ملف `vercel.json` مع `rewrites` لـ API
- [ ] تحديث `API_URL` ليقرأ من متغير بيئة `VITE_API_URL`
- [ ] CI/CD: نشر تلقائي عند push لـ main

#### 8C — المصادقة وتعدد المستخدمين
- [ ] تسجيل دخول بكلمة مرور (JWT)
- [ ] أدوار: مدير / وكيل / قراءة فقط
- [ ] كل وكيل يرى المحادثات المعيّنة له فقط
- [ ] سجل نشاط (من فعل ماذا ومتى)

### المرحلة 9 — الذكاء الاصطناعي المتقدم
**الأولوية:** متوسطة | **التعقيد:** كبير
